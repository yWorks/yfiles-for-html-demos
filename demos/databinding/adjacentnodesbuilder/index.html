Description

Interactively create and modify a graph from business data with class yfiles.binding.AdjacentNodesGraphBuilder.

yfiles.binding.AdjacentNodesGraphBuilder stores the business data object of each node in the node's yfiles.graph.ITagOwner#tag property, thus, the properties of a node's yfiles.styles.TemplateNodeStyle can be bound to this object.

Things to Try

  • Select a sample in the combo box and inspect its data in the settings that define the graph.
  • Add or modify nodes in the Nodes Source.
  • Add or modify predecessors and successors in the Nodes Source.
  • Modify the SVG code for the Node Template.
  • Build a graph for your own data and bindings.

Graph Source Settings

  • While New Graph creates a new graph from scratch, Update Graph adjusts the graph to changed data and keeps nodes and edges whose source objects are still present.
  • Nodes Source contains the data that specifies the graph's nodes. This can either be an array or another object containing the node business data.
  • The Predecessors (Successors) Binding specifies the predecessors (successors) of the node. Similar to the data of Nodes Source, the resolved data can either be an array or another object containing the node business data. This binding itself can either be the path to the property specifying the data, or a function that returns the data.
  • The Node ID Binding is an optional property to specify a property path that returns a node's ID. If present, the node ID has to be used in the predecessors and successors bindings.
  • Node Template defines the SVG template that is used to visualize a node. This may contain dynamic bindings as demonstrated in the samples that allow visualizing arbitrary properties of the node's business data, such as the name or id properties.
