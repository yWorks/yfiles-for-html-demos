Description

This demo loads the yFiles modules using <script> tags instead of an AMD loader.

With script-tags, dependencies between the yFiles modules are not resolved automatically. Therefore, all modules that are used by an application have to be loaded separately and in the correct order, as shown in this yFiles modules dependency diagram.

Deployment Tool

yFiles comes with a deployment tool that can combine the yFiles modules into fewer or optionally even a single file. See the tool's documentation for details.
