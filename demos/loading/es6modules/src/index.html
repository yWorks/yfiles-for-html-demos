Description

This demo loads the yFiles module resources using ES6 module imports.

Using the ES6 library modules, you can load the yFiles types through import statements , e.g.:

import { GraphComponent, GraphEditorInputMode, Rect, ICommand, TimeSpan } from '../../../lib/es6-modules/yfiles/view-editor.js';

Additionally, this demo includes a gruntfile that showcases how the yFiles deployment tool can be used to create a minified and optimized version of this demo.

This includes an optional bundling step that is performed by rollup.js

Using the gruntfile

First, make sure that the required npm modules are installed in the demo directory:

> npm install

To create the minified and optimized version, simply run the deploy script:

> npm run deploy

The output will be located in the ./deployed folder. You can look at the result by opening the index.html file in that folder.

Optionally, you can run the bundle script:

> npm run bundle

This will use rollup.js to create a single .js file that contains all the dependencies of the demo. The created bundle will be located in the ./bundled folder.
